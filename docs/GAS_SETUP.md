# GAS（Google Apps Script）連携セットアップ手順

予約フォーム → GAS → スプレッドシート保存 → Gmail通知 → 自動返信、の一連の流れを
有効にするための手順です。所要時間は約10分です。

## 1. Googleスプレッドシートを新規作成

1. Google Drive で新しいスプレッドシートを作成する（例：「il PhotoWedd 予約管理」）
2. シート名は何でもよい（GASが自動で「予約管理」シートを作成します）

## 2. Apps Scriptを紐付ける

1. スプレッドシートのメニューから **拡張機能 → Apps Script** を開く
2. デフォルトの `Code.gs` の内容を全て削除し、このリポジトリの
   [`gas/Code.gs`](../gas/Code.gs) の内容を貼り付ける
3. `CONFIG.STAFF_EMAIL` が通知を受け取りたいメールアドレスになっているか確認する
   （現在の設定：`enchanteyokohama@gmail.com`）

## 3. 権限を許可する（重要）

Webアプリとして匿名ユーザーからの送信を受け付けるには、事前にオーナー自身が
一度スクリプトを実行して権限（スプレッドシート・Gmail送信）を許可しておく必要があります。

1. Apps Scriptエディタの関数選択で `doGet` を選び、▶ 実行 をクリック
2. 「権限を確認」画面が出たら、自分のGoogleアカウントを選択し「許可」する
3. 実行ログに `{"status":"ok", ...}` が表示されれば成功

## 4. ウェブアプリとしてデプロイ

1. 右上の **デプロイ → 新しいデプロイ** をクリック
2. 種類の選択で「ウェブアプリ」を選ぶ
3. 設定：
   - 説明：`reservation-form-v1`（任意）
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
4. 「デプロイ」をクリックし、発行された **ウェブアプリのURL**（`.../exec` で終わるURL）をコピーする

## 5. LP側にURLを設定

1. [`js/main.js`](../js/main.js) を開く
2. 先頭付近の以下の行を、コピーしたURLに書き換える

   ```js
   const GAS_ENDPOINT = 'https://script.google.com/macros/s/REPLACE_WITH_YOUR_DEPLOYMENT_ID/exec';
   ```

3. 保存してコミットする

## 6. 動作確認

1. LPをブラウザで開き、予約フォームに入力して送信する
2. 以下がすべて成功すればOK：
   - `thanks.html` に遷移し、予約番号が表示される
   - スプレッドシートの「予約管理」シートに行が追加される
   - `enchanteyokohama@gmail.com` にスタッフ通知メールが届く
   - フォームに入力したメールアドレスに自動返信メールが届く

## コードを更新した場合

`Code.gs` を編集したあとは、**デプロイ → デプロイを管理 → 編集(鉛筆アイコン) →
バージョン：新しいバージョン → デプロイ** を行わないと、公開URLの動作は更新されません。
`main.js` 側のURLは変わらないので書き換え不要です。

## シート列構成

| 列 | 内容 |
|---|---|
| 受付日時 | フォーム送信日時（自動記録） |
| 予約番号 | `IL` + 送信日 + 連番（自動発行） |
| 氏名 / フリガナ | フォーム入力値 |
| メール / 電話 | フォーム入力値 |
| プラン | 希望プラン |
| 相談方法 | LINE / オンライン / 電話 / メール |
| 希望日時 | 撮影・相談の希望日 |
| 備考 | ご質問・ご要望（自由記述） |
| 対応状況 | 初期値「未対応」。スタッフが手動で更新する運用列 |

## トラブルシューティング

- **送信してもメールが届かない** → 手順3の権限許可が実行されているか確認
- **フォーム送信でエラーになる** → デプロイ設定の「アクセスできるユーザー」が「全員」になっているか確認
- **シートに行が増えない** → `CONFIG.SHEET_NAME` を変更していないか、シートが誤って削除されていないか確認
